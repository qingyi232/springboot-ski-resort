## 二、部署步骤

### 步骤1：初始化数据库

1. 启动MySQL服务
2. 打开命令行或Navicat，执行SQL脚本：
```bash
mysql -u root -p < backend\sql\initdatabase.sql
```
> 如果MySQL密码不是 `root`，需要修改 `backend\src\main\resources\application.yml` 中的 `spring.datasource.password`

### 步骤2：启动Redis

双击 `redis-server.exe` 或在cmd中运行：
```bash
redis-server
```

### 步骤3：打包后端

在项目 `backend` 目录下执行：
```bash
mvn clean package -DskipTests
```
打包完成后会在 `backend\target\` 下生成 `ski-resort-backend-1.0.0.jar`

### 步骤4：构建前端

在项目 `frontend` 目录下执行：
```bash
npm install
npm run build
```
构建完成后会在 `frontend\dist\` 下生成静态文件

### 步骤5：配置Nginx

将 `frontend\dist\` 文件夹复制到 Nginx 安装目录下，
然后将 `nginx.conf` 替换为项目提供的配置（见下方 `nginx.conf` 文件）

### 步骤6：启动所有服务

使用项目根目录下的 `启动系统.bat` 一键启动：
```
双击 启动系统.bat
```

## 三、配置修改说明

### 3.1 如果MySQL密码不同
修改 `application.yml`：
```yaml
spring:
  datasource:
    password: 你的密码
```

### 3.2 如果端口被占用
- 后端默认端口：**8080**（在 `application.yml` 中 `server.port` 修改）
- Nginx默认端口：**80**（在 `nginx.conf` 中修改）

## 五、常见问题

**Q: 后端启动报数据库连接失败？**  
A: 确认MySQL服务已启动，数据库 `ski_resort` 已创建，用户名密码正确。

**Q: 前端页面刷新后404？**  
A: 检查Nginx配置中 `try_files` 是否配置正确，Vue使用了HTML5 History模式。

**Q: Redis连接失败？**  
A: 确认Redis服务已启动。如果不需要缓存功能，Redis连接失败不会影响主流程（只是缓存不生效）。

**Q: 启动后端报端口占用？**  
A: 在cmd中执行 `netstat -ano | findstr :8080` 找到占用进程，用 `taskkill /PID xxx /F` 结束。
