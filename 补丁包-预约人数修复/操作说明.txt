===================================================
  飞跃滑雪场管理系统 - 预约人数Bug修复补丁
  修复日期：2026-05-19
===================================================

【问题描述】
用户预约场地时填写的预约人数（如7人），管理员端看到的却是1人。
同时管理员端预订详情的"时长""单价""场地类型""难度级别"也显示异常。

【修复内容】
修改了 3 个文件，修复了预约人数不保存、管理端字段名不匹配等问题。

===================================================
  文件覆盖指南（共3个文件）
===================================================

请按照下面的路径，将补丁包中的文件覆盖到你项目中对应的位置。
假设你的项目根目录在 D:\xxxx\基于springboot的飞跃滑雪场管理系统的设计与实现

---

【文件1】VenueBookingMapper.xml（后端 Mapper 文件）

  补丁包位置：
    补丁包-预约人数修复\backend\src\main\resources\mapper\VenueBookingMapper.xml

  覆盖到项目路径：
    你的项目\backend\src\main\resources\mapper\VenueBookingMapper.xml

  修改说明：
    - INSERT语句添加了 people_count 字段（核心修复：预约人数现在会正确保存）
    - UPDATE语句添加了 people_count 字段
    - 查询语句增加了LEFT JOIN场地表，返回场地类型、难度等级等信息
    - 通用查询列添加了 people_count

  覆盖到项目路径：

【文件3】BookingManagement.vue（前端管理页面）

  补丁包位置：

  覆盖到项目路径：
    你的项目\frontend\src\views\admin\BookingManagement.vue

  修改说明：
    - "时长"字段从 duration 改为 hours（修复时长显示为空的问题）
    - "单价"字段从 hourlyRate 改为 rentalPrice（修复单价显示为0的问题）
    - "难度级别"字段从 difficultyLevel 改为 venueDifficulty（修复难度显示为-的问题）

===================================================
  覆盖后的操作步骤
===================================================

1. 关闭 IntelliJ IDEA 中正在运行的后端服务

2. 按照上面的路径覆盖3个文件

3. 在 IDEA 中重新打开项目，等待自动编译完成
   （如果没有自动编译，按 Ctrl+F9 手动编译）

4. 重新启动后端服务（运行 SpringBoot 主类）

5. 前端需要重新打包：
   - 打开命令行，进入项目的 frontend 目录
   - 运行: npm run build
   - 将生成的 dist 文件夹部署到服务器

6. 刷新浏览器页面，测试预约功能
   - 用户端：预约场地时填写人数（如5人）
   - 管理员端：查看该预订详情，确认人数显示为5人

===================================================
  注意事项
===================================================

- 之前已有的预约记录人数仍然显示为1（因为之前没有保存人数）
- 只有新的预约记录才会正确保存和显示预约人数
- 如果覆盖前想备份，先把原文件复制一份再覆盖
